# Extended Contributing Guide
## Branch Naming
- feat/your-feature  for new features
- fix/bug-name       for bug fixes
- docs/topic         for documentation
- chore/task         for maintenance
## Commit Convention
Follow Conventional Commits: feat:, fix:, docs:, chore:, refactor:, test:
## PR Process
1. Fork and clone, create branch, commit, push, open PR
2. Fill in the PR template, request review
3. Squash merge after approval
